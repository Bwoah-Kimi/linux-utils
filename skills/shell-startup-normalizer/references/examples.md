# Examples

## Bash Migration With Existing Split Files

Before:

```bash
# ~/.bashrc
export PATH="$HOME/.local/bin:$PATH"
alias gs='git status'

proxy() {
  ~/.local/bin/proxy "$@"
}

eval "$(oh-my-posh init bash)"
```

After:

```bash
# ~/.bashrc
case $- in
  *i*) ;;
  *) return ;;
esac

[ -f "$HOME/.shell_env" ] && . "$HOME/.shell_env"
[ -f "$HOME/.shell_aliases" ] && . "$HOME/.shell_aliases"
[ -f "$HOME/.shell_functions" ] && . "$HOME/.shell_functions"

eval "$(oh-my-posh init bash)"
[ -f "$HOME/.shell_local" ] && . "$HOME/.shell_local"
```

```bash
# ~/.shell_aliases
alias gs='git status'
```

```bash
# ~/.shell_functions
proxy() {
  ~/.local/bin/proxy "$@"
}
```

## Messy Single-File Zsh Migration

Before:

```zsh
export PATH="$HOME/bin:/opt/homebrew/bin:$PATH"
alias ll='ls -lah'
autoload -U compinit && compinit
eval "$(starship init zsh)"
```

After:

```zsh
# ~/.zprofile
[ -f "$HOME/.shell_env" ] && . "$HOME/.shell_env"
```

```zsh
# ~/.zshrc
[ -f "$HOME/.shell_env" ] && . "$HOME/.shell_env"
[ -f "$HOME/.shell_aliases" ] && . "$HOME/.shell_aliases"
[ -f "$HOME/.shell_functions" ] && . "$HOME/.shell_functions"

autoload -U compinit && compinit
eval "$(starship init zsh)"
[ -f "$HOME/.shell_local" ] && . "$HOME/.shell_local"
```

```bash
# ~/.shell_aliases
alias ll='ls -lah'
```

## Preserve Tool-Managed Blocks

Keep generated blocks intact, even if you move them to a more appropriate file:

```bash
# >>> conda initialize >>>
# ... generated by conda ...
# <<< conda initialize <<<
```

```bash
eval "$(oh-my-posh init bash)"
```

The wrapper files around those blocks can be cleaned up. The generated internals should not be rewritten unless the user asks.
